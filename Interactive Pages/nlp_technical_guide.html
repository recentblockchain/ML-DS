🎯 What is Natural Language Processing?

Definition

Natural Language Processing (NLP) is the field of AI that enables computers to understand, interpret, and generate human language. It bridges the gap between human communication and computer understanding.

Core Challenges

  • Ambiguity: "Time flies like an arrow" vs "Fruit flies like a banana"
  • Context: "Bank" (financial institution vs river bank)
  • Variability: Same meaning, different words ("happy" vs "joyful")
  • World Knowledge: "The trophy doesn't fit in the suitcase because it's too big" (what is "it"?)

Applications

  • Machine Translation (Google Translate)
  • Sentiment Analysis (Product reviews)
  • Chatbots & Virtual Assistants (Siri, Alexa)
  • Information Extraction (Named Entity Recognition)
  • Question Answering
  • Text Summarization
  • Text Generation (GPT models)

📝 Text Representation: From Words to Numbers

1. Tokenization

Breaking Text into Units

Tokenization is the process of breaking text into smaller units (tokens) - typically words or subwords.

Sentence: "Natural Language Processing is amazing!"

Tokens:

Natural Language Processing is amazing !

Tokenization Strategies

Method Description Example Use Case
Word-level Split by whitespace/punctuation "Hello world" → ["Hello", "world"] Simple tasks
Character-level Individual characters "Hello" → ["H", "e", "l", "l", "o"] Spelling, morphology
Subword (BPE) Between word and character "unhappiness" → ["un", "happi", "ness"] Modern transformers
SentencePiece Language-independent subwords Handles all languages uniformly Multilingual models

🔄 Sequential Models for Text

1. Recurrent Neural Networks (RNN)

Processing Sequences Step by Step

RNNs process text sequentially, maintaining a hidden state that carries information from previous words.

RNN Mathematics

hₜ = tanh(Wₕₕ·hₜ₋₁ + Wₓₕ·xₜ + bₕ)
yₜ = Wₕᵧ·hₜ + bᵧ

where:
hₜ = hidden state at time t
xₜ = input at time t
W = weight matrices
b = bias vectors

Problems with Basic RNNs

  • Vanishing Gradients: Difficult to learn long-term dependencies
  • Exploding Gradients: Gradients can grow exponentially
  • Short Memory: Struggles with sequences longer than 10-20 steps

2. Long Short-Term Memory (LSTM)

Solving the Long-Term Dependency Problem

LSTMs use gating mechanisms to selectively remember and forget information over long sequences.

LSTM Cell Structure

Forget Gate
fₜ = σ(Wf·[hₜ₋₁, xₜ])
Input Gate
iₜ = σ(Wi·[hₜ₋₁, xₜ])
Output Gate
oₜ = σ(Wo·[hₜ₋₁, xₜ])
↓
Cell State Update
Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ
↓
Hidden State
hₜ = oₜ ⊙ tanh(Cₜ)

LSTM Gates Explained

  • Forget Gate (fₜ): Decides what information to discard from cell state
  • Input Gate (iₜ): Decides what new information to add to cell state
  • Output Gate (oₜ): Decides what to output based on cell state
  • Cell State (Cₜ): The "memory" that flows through the network

✅ LSTM Advantages

  • Handles sequences of 100+ tokens effectively
  • Mitigates vanishing gradient problem
  • Can learn what to remember and what to forget
  • State-of-the-art for many sequence tasks (2015-2018)

👁️ Attention Mechanism

The Breakthrough That Changed NLP

Attention allows the model to focus on relevant parts of the input sequence when producing each output. Instead of compressing the entire input into a fixed vector, attention computes weighted combinations of all input positions.

Example: Translating "The cat sat on the mat"

When translating "cat" to French ("chat"), the model pays high attention to "cat" and moderate attention to "The":

The
cat
sat
on

Attention weights when generating "chat"

Attention Formula

Attention(Q, K, V) = softmax(Q·Kᵀ / √dₖ)·V

Q = Query (what we're looking for)
K = Key (what each position represents)
V = Value (what information to extract)
dₖ = dimension of keys (for scaling)

Types of Attention

  • Self-Attention: Relate different positions of the same sequence
  • Cross-Attention: Relate positions between two sequences (encoder-decoder)
  • Multi-Head Attention: Multiple attention mechanisms in parallel

⚡ Transformers: The Modern Standard

"Attention Is All You Need" (2017)

Transformers replaced RNNs by using only attention mechanisms, enabling parallel processing and much longer context windows.

Key Innovations

  • Parallel Processing: No sequential dependency like RNNs (much faster training)
  • Long-Range Dependencies: Attention to any position directly
  • Positional Encoding: Adds position information to embeddings
  • Multi-Head Attention: Multiple attention patterns simultaneously

Generation Tasks (Translation, Summarization)

  • BLEU: Precision of n-grams (0-1, higher better) - Used for translation
  • ROUGE: Recall of n-grams (0-1, higher better) - Used for summarization
  • Perplexity: How surprised model is by text (lower better) - Language modeling
  • BERTScore: Semantic similarity using BERT embeddings

Question Answering

  • Exact Match (EM): Percentage of predictions exactly matching ground truth
  • F1-Score: Token-level overlap between prediction and ground truth

⚠️ Current Challenges

Open Problems in NLP

  • Common Sense Reasoning: Models lack real-world knowledge
  • Factual Accuracy: Hallucinations and false information
  • Bias & Fairness: Models inherit biases from training data
  • Low-Resource Languages: Most models focused on English
  • Interpretability: Difficult to understand why models make decisions
  • Computational Cost: Training large models requires massive resources
  • Context Length: Limited to few thousand tokens (improving)

📝 Summary

Key Takeaways

  • Text Representation: Words → Tokens → Embeddings → Dense vectors
  • Sequential Models: RNN → LSTM/GRU → Transformers
  • Attention: Revolutionary mechanism enabling long-range dependencies
  • Transformers: Parallel processing, self-attention, state-of-the-art
  • BERT: Bidirectional, best for understanding tasks
  • GPT: Autoregressive, best for generation tasks
  • Modern NLP: Pre-training + Fine-tuning paradigm
  • Performance: Near-human level on many benchmarks
