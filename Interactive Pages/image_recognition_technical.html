2. Convolutional Neural Network (CNN) Architecture

🧠 Why CNNs for Images?

Problem with Regular Neural Networks:

  • 224×224×3 image = 150,528 input features
  • Even small hidden layer with 1000 neurons = 150 million parameters!
  • Completely ignores spatial structure of images

CNN Solution:

  • Local connectivity: Each neuron connects to small region
  • Parameter sharing: Same weights used across entire image
  • Spatial hierarchy: Early layers detect edges, later layers detect objects

Understanding Convolution Operation

🔍 What is Convolution?

Convolution slides a small filter (kernel) across the image, computing dot products at each position.

Mathematical Definition: Output(i, j) = Σ Σ Input(i+m, j+n) × Kernel(m, n) m n For a 3×3 kernel: - Input: Local image patch (3×3) - Kernel: Learned weights (3×3) - Output: Single number (activation) Example: Input patch: Kernel (Edge Detector): [1 2 3] [-1 -1 -1] [4 5 6] × [ 0 0 0] = -12 [7 8 9] [ 1 1 1] Calculation: = 1×(-1) + 2×(-1) + 3×(-1) + 4×0 + 5×0 + 6×0 + 7×1 + 8×1 + 9×1 = -1 - 2 - 3 + 0 + 0 + 0 + 7 + 8 + 9 = -12
Convolution from Scratch Python
import numpy as np def convolution_2d(image, kernel): """ Perform 2D convolution Args: image: Input image (H, W) kernel: Convolution kernel (K, K) Returns: Convolved output """ # Get dimensions img_h, img_w = image.shape ker_h, ker_w = kernel.shape # Calculate output dimensions out_h = img_h - ker_h + 1 out_w = img_w - ker_w + 1 # Initialize output output = np.zeros((out_h, out_w)) # Perform convolution for i in range(out_h): for j in range(out_w): # Extract patch patch = image[i:i+ker_h, j:j+ker_w] # Compute dot product output[i, j] = np.sum(patch * kernel) return output # Demo: Edge detection print("=== CONVOLUTION DEMO ===") # Create sample image (gradient) image = np.array([ [0, 0, 0, 255, 255], [0, 0, 0, 255, 255], [0, 0, 0, 255, 255], [0, 0, 0, 255, 255], [0, 0, 0, 255, 255] ], dtype=np.float32) # Different kernels kernels = { 'Vertical Edge': np.array([ [-1, 0, 1], [-1, 0, 1], [-1, 0, 1] ]), 'Horizontal Edge': np.array([ [-1, -1, -1], [ 0, 0, 0], [ 1, 1, 1] ]), 'Sharpen': np.array([ [ 0, -1, 0], [-1, 5, -1], [ 0, -1, 0] ]) } for name, kernel in kernels.items(): result = convolution_2d(image, kernel) print(f"\n{name} Detector:") print(f"Output shape: {result.shape}") print(f"Output:\n{result.astype(int)}")

Understanding Pooling

🔽 Why Pooling?

Purpose: Reduce spatial dimensions while retaining important features

Benefits:

  • Reduces computation (fewer parameters)
  • Provides translation invariance
  • Controls overfitting
Max Pooling (2×2): Input (4×4): Output (2×2): [1 3 2 4] [6 8] [5 6 1 2] → [9 7] [2 9 4 3] [1 0 7 2] Each 2×2 region → Maximum value Top-left: max(1,3,5,6) = 6 Top-right: max(2,4,1,2) = 8 Bottom-left: max(2,9,1,0) = 9 Bottom-right: max(4,3,7,2) = 7
Pooling from Scratch Python
def max_pooling_2d(image, pool_size=2, stride=2): """ Perform max pooling Args: image: Input feature map (H, W) pool_size: Size of pooling window stride: Step size Returns: Pooled output """ h, w = image.shape # Calculate output dimensions out_h = (h - pool_size) // stride + 1 out_w = (w - pool_size) // stride + 1 output = np.zeros((out_h, out_w)) # Perform pooling for i in range(out_h): for j in range(out_w): # Extract window h_start = i * stride w_start = j * stride window = image[h_start:h_start+pool_size, w_start:w_start+pool_size] # Take maximum output[i, j] = np.max(window) return output # Demo print("=== POOLING DEMO ===") feature_map = np.array([ [1, 3, 2, 4], [5, 6, 1, 2], [2, 9, 4, 3], [1, 0, 7, 2] ], dtype=np.float32) print(f"Input shape: {feature_map.shape}") print(f"Input:\n{feature_map.astype(int)}") pooled = max_pooling_2d(feature_map, pool_size=2, stride=2) print(f"\nOutput shape: {pooled.shape}") print(f"Output:\n{pooled.astype(int)}") print(f"\nSize reduction: {feature_map.size} → {pooled.size} pixels")

3. Building a Complete CNN from Scratch

CNN for MNIST Digit Recognition Python
import tensorflow as tf from tensorflow import keras from tensorflow.keras import layers import numpy as np import matplotlib.pyplot as plt print("=== BUILDING CNN FOR MNIST ===") # Load MNIST dataset (x_train, y_train), (x_test, y_test) = keras.datasets.mnist.load_data() print(f"\nDataset Info:") print(f" Training samples: {x_train.shape[0]:,}") print(f" Test samples: {x_test.shape[0]:,}") print(f" Image size: {x_train.shape[1]}×{x_train.shape[2]}") print(f" Classes: {len(np.unique(y_train))}") # Preprocessing x_train = x_train.astype("float32") / 255.0 x_test = x_test.astype("float32") / 255.0 # Reshape for CNN (add channel dimension) x_train = np.expand_dims(x_train, -1) x_test = np.expand_dims(x_test, -1) print(f"\nAfter preprocessing: {x_train.shape}") # Build CNN architecture model = keras.Sequential([ # Layer 1: Convolution + ReLU + Pooling layers.Conv2D(32, kernel_size=(3, 3), activation='relu', input_shape=(28, 28, 1), name='conv1'), layers.MaxPooling2D(pool_size=(2, 2), name='pool1'), # Layer 2: Convolution + ReLU + Pooling layers.Conv2D(64, kernel_size=(3, 3), activation='relu', name='conv2'), layers.MaxPooling2D(pool_size=(2, 2), name='pool2'), # Flatten for fully connected layers layers.Flatten(name='flatten'), # Fully connected layers layers.Dropout(0.5, name='dropout'), layers.Dense(128, activation='relu', name='fc1'), layers.Dense(10, activation='softmax', name='output') ], name='MNIST_CNN') # Display architecture print("\n=== MODEL ARCHITECTURE ===") model.summary() # Compile model model.compile( optimizer='adam', loss='sparse_categorical_crossentropy', metrics=['accuracy'] ) print("\n✓ Model compiled")

🎓 Key Insights

  • Hierarchical Learning: Early layers detect simple features (edges), deeper layers detect complex patterns (shapes)

  • Dimensionality Reduction: Pooling reduces spatial size while keeping important information

5. Transfer Learning with Pre-trained Models

🚀 Why Transfer Learning?

Problem: Training CNNs from scratch requires millions of images and days/weeks of GPU time

Solution: Use pre-trained models (trained on ImageNet) and fine-tune for your task

Benefits:

  • Requires far less data (hundreds vs millions of images)
  • Faster training (hours vs days)
  • Often achieves better performance
Transfer Learning Example Python
from tensorflow.keras.applications import VGG16 from tensorflow.keras.applications.vgg16 import preprocess_input, decode_predictions from tensorflow.keras.preprocessing import image print("=== TRANSFER LEARNING WITH VGG16 ===") # Load pre-trained VGG16 model print("\n1. ({len(base_model.layers)-7} more layers) ...") # Use for prediction print("\n3. Making Predictions...") # Load and preprocess image img_path = 'cat.jpg' img = image.load_img(img_path, target_size=(224, 224)) x = image.img_to_array(img) x = np.expand_dims(x, axis=0) x = preprocess_input(x) # Predict import time start_time = time.time() preds = base_model.predict(x, verbose=0) inference_time = (time.time() - start_time) * 1000 # Decode predictions top_predictions = decode_predictions(preds, top=5)[0] print(f"\n Top 5 Predictions:") for i, (imagenet_id, label, score) in enumerate(top_predictions, 1): print(f" {i}. {label:20s} {score*100:5.2f}%") print(f"\n Inference time: {inference_time:.1f}ms")

6. Production Best Practices

⚡ Performance Optimization

  • Use GPU for training/inference
  • Batch processing for throughput
  • Model quantization for deployment
  • TensorRT for NVIDIA GPUs

🎯 Data Augmentation

  • Random rotation (±15°)
  • Horizontal flip
  • Zoom in/out (10-20%)
  • Brightness adjustment

🛡️ Preventing Overfitting

  • Dropout (0.3-0.5)
  • L2 regularization
  • Early stopping
  • More training data

📊 Model Selection

  • MobileNet: Mobile/edge
  • ResNet: General purpose
  • EfficientNet: Best accuracy
  • YOLO: Object detection

Complete Training Pipeline

Production-Ready Training Code Python
from tensorflow.keras.preprocessing.image import ImageDataGenerator from tensorflow.keras.callbacks import EarlyStopping, ModelCheckpoint print("=== PRODUCTION TRAINING PIPELINE ===") # 1. Data Augmentation print("\n1. Setting up data augmentation...") train_datagen = ImageDataGenerator( rotation_range=15, width_shift_range=0.1, height_shift_range=0.1, zoom_range=0.1, horizontal_flip=True, fill_mode='nearest' ) # 2. Callbacks print("2. Configuring callbacks...") early_stop = EarlyStopping( monitor='val_loss', patience=5, restore_best_weights=True ) checkpoint = ModelCheckpoint( 'best_model.h5', monitor='val_accuracy', save_best_only=True ) # 3. Training print("3. Training with augmentation...") history = model.fit( train_datagen.flow(x_train, y_train, batch_size=128), epochs=50, validation_data=(x_test, y_test), callbacks=[early_stop, checkpoint], verbose=1 ) print(f"\n✓ Training completed") print(f" Best validation accuracy: {max(history.history['val_accuracy'])*100:.2f}%") print(f" Total epochs: {len(history.history['loss'])}")

✅ Summary: CNN for Image Recognition

  • Core Concept: CNNs learn hierarchical features automatically from data
  • Key Operations: Convolution (feature extraction) + Pooling (dimensionality reduction)
  • Architecture: Stack conv-pool blocks, then flatten to dense layers
  • Training: Use data augmentation, callbacks, and regularization
  • Transfer Learning: Fine-tune pre-trained models for faster, better results
  • Performance: Modern CNNs achieve >95% accuracy on complex tasks
