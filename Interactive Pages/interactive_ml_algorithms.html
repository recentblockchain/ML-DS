📈 Linear Regression

What it does: Finds the best-fit line through data points to predict continuous values.
Use cases: House price prediction, sales forecasting, trend analysis

How It Works

Linear regression tries to find the line y = mx + b that minimizes the distance between the line and all data points. It uses gradient descent to optimize the slope (m) and intercept (b).

Python Implementation

# Linear Regression from scratch import numpy as np class LinearRegression: def __init__(self, learning_rate=0.01, iterations=1000): self.lr = learning_rate self.iterations = iterations self.slope = 0 self.intercept = 0 def fit(self, X, y): n = len(X) for _ in range(self.iterations): # Make predictions y_pred = self.slope * X + self.intercept # Calculate gradients d_slope = (-2/n) * sum(X * (y - y_pred)) d_intercept = (-2/n) * sum(y - y_pred) # Update parameters self.slope -= self.lr * d_slope self.intercept -= self.lr * d_intercept def predict(self, X): return self.slope * X + self.intercept # Using scikit-learn from sklearn.linear_model import LinearRegression model = LinearRegression() model.fit(X_train, y_train) predictions = model.predict(X_test) # Example: Predicting house prices X = np.array([[1500], [2000], [2500]]) # Square footage y = np.array([300000, 400000, 500000]) # Price model = LinearRegression() model.fit(X, y) predicted_price = model.predict([[2200]]) # Predict for 2200 sq ft
Real Example: Predicting student test scores based on study hours
  • Input: Hours studied (2, 3, 5, 7, 9)
  • Output: Test scores (50, 60, 75, 85, 95)
  • Model learns: More study hours → Higher scores

🎯 K-Means Clustering

What it does: Groups similar data points into K clusters without labels.
Use cases: Customer segmentation, image compression, anomaly detection

How It Works

K-Means iteratively assigns points to the nearest cluster center, then recalculates centers based on assigned points. It repeats until centers stop moving.

Python Implementation

# K-Means from scratch import numpy as np class KMeans: def __init__(self, k=3, max_iters=100): self.k = k self.max_iters = max_iters self.centroids = None def fit(self, X): # Randomly initialize centroids random_idx = np.random.choice(len(X), self.k, replace=False) self.centroids = X[random_idx] for _ in range(self.max_iters): # Assign points to nearest centroid labels = self._assign_clusters(X) # Update centroids new_centroids = np.array([ X[labels == i].mean(axis=0) for i in range(self.k) ]) # Check convergence if np.all(self.centroids == new_centroids): break self.centroids = new_centroids return labels def _assign_clusters(self, X): distances = np.sqrt(((X - self.centroids[:, np.newaxis])**2).sum(axis=2)) return np.argmin(distances, axis=0) # Using scikit-learn from sklearn.cluster import KMeans kmeans = KMeans(n_clusters=3, random_state=42) labels = kmeans.fit_predict(X) # Example: Customer segmentation # X = customer features (age, spending, visits) # Groups customers into 3 segments automatically
Real Example: Customer Segmentation for E-commerce
  • Input: Customer data (purchase frequency, average spend)
  • K=3 clusters might reveal: Budget shoppers, Regular customers, VIP customers
  • Use: Targeted marketing campaigns for each segment

👥 K-Nearest Neighbors (KNN)

What it does: Classifies data based on the majority vote of its K nearest neighbors.
Use cases: Recommendation systems, pattern recognition, medical diagnosis

How It Works

KNN is simple: "You are who your neighbors are." To classify a new point, it looks at the K closest training points and assigns the most common class.

Python Implementation

# KNN from scratch import numpy as np from collections import Counter class KNN: def __init__(self, k=3): self.k = k def fit(self, X, y): self.X_train = X self.y_train = y def predict(self, X): predictions = [self._predict_single(x) for x in X] return np.array(predictions) def _predict_single(self, x): # Calculate distances to all training points distances = [np.sqrt(np.sum((x - x_train)**2)) for x_train in self.X_train] # Get K nearest neighbors k_indices = np.argsort(distances)[:self.k] k_nearest_labels = [self.y_train[i] for i in k_indices] # Return most common label most_common = Counter(k_nearest_labels).most_common(1) return most_common[0][0] # Using scikit-learn from sklearn.neighbors import KNeighborsClassifier knn = KNeighborsClassifier(n_neighbors=5) knn.fit(X_train, y_train) predictions = knn.predict(X_test) # Example: Movie recommendation # Find 5 users similar to you, recommend movies they liked
Real Example: Medical Diagnosis
  • Input: Patient symptoms (fever level, blood pressure, age)
  • Compare to K=5 most similar past patients
  • Diagnose based on what those 5 patients had

⚔️ Support Vector Machine (SVM)

What it does: Finds the best boundary (hyperplane) that separates different classes with maximum margin.
Use cases: Text classification, image recognition, bioinformatics

How It Works

SVM finds the line (or hyperplane in higher dimensions) that separates classes with the widest possible margin. Points closest to the boundary are called "support vectors."

📊 Naive Bayes

What it does: Uses Bayes' theorem to calculate probability of each class given the features.
Use cases: Text classification, spam filtering, sentiment analysis

How It Works

Naive Bayes calculates: P(Class|Features) using Bayes' theorem. It's "naive" because it assumes all features are independent (which often isn't true, but it works well anyway!).

Visual Example: Email Classification

Training Data:

Email Text Contains "free" Contains "meeting" Class
Free money now! ✓ ✗ Spam
Meeting at 3pm ✗ ✓ Not Spam
Free meeting room ✓ ✓ Not Spam

Test: "Free discount meeting"

Calculation:

P(Spam | "free", "meeting") vs P(Not Spam | "free", "meeting")

Model counts how often "free" and "meeting" appear in each class, then picks the more likely class.

Python Implementation

# Gaussian Naive Bayes from scratch import numpy as np class GaussianNB: def fit(self, X, y): self.classes = np.unique(y) self.mean = {} self.var = {} self.priors = {} for c in self.classes: X_c = X[y == c] self.mean[c] = X_c.mean(axis=0) self.var[c] = X_c.var(axis=0) self.priors[c] = len(X_c) / len(X) def _gaussian_probability(self, x, mean, var): exp = np.exp(-((x - mean)**2) / (2 * var)) return (1 / np.sqrt(2 * np.pi * var)) * exp def predict(self, X): predictions = [] for x in X: posteriors = [] for c in self.classes: prior = np.log(self.priors[c]) likelihood = np.sum(np.log( self._gaussian_probability(x, self.mean[c], self.var[c]) )) posteriors.append(prior + likelihood) predictions.append(self.classes[np.argmax(posteriors)]) return np.array(predictions) # Using scikit-learn from sklearn.naive_bayes import GaussianNB, MultinomialNB # For continuous features gnb = GaussianNB() gnb.fit(X_train, y_train) # For text classification (word counts) from sklearn.feature_extraction.text import CountVectorizer vectorizer = CountVectorizer() X_text = vectorizer.fit_transform(emails) mnb = MultinomialNB() mnb.fit(X_text, labels) predictions = mnb.predict(vectorizer.transform(new_emails))

  • Features: Word frequencies (happy, terrible, love, hate)
  • Model learns: Which words are more common in positive vs negative reviews
  • New review: Calculate probability based on its words

🌳 Decision Tree

What it does: Creates a tree of if-then rules to make decisions.
Use cases: Medical diagnosis, credit approval, customer churn prediction

How It Works

A decision tree asks a series of yes/no questions about features, splitting the data at each step to create the purest groups possible. It uses metrics like Gini impurity or Information Gain to decide the best splits.

Visual Example: Should I Play Tennis?

🌳 Decision Tree
Outlook?
├─ Sunny → Humidity?
    ├─ High → ❌ Don't Play
    └─ Normal → ✅ Play
├─ Overcast → ✅ Play
└─ Rain → Windy?
    ├─ Yes → ❌ Don't Play
    └─ No → ✅ Play

How It's Built:

  1. Start with all data at the root
  2. Find the feature that best splits the data (highest information gain)
  3. Split the data based on that feature
  4. Repeat for each branch until data is pure or stopping criteria is met

🌲🌳🌲 Random Forest

What it does: Creates many decision trees and combines their predictions (ensemble learning).
Use cases: Credit scoring, fraud detection, disease prediction

How It Works

Random Forest builds multiple decision trees, each trained on a random subset of data and features. For predictions, all trees vote and the majority wins. This reduces overfitting and improves accuracy.

Visual Explanation

🌳 Tree 1
Vote: ✅
🌳 Tree 2
Vote: ✅
🌳 Tree 3
Vote: ❌
🌳 Tree 4
Vote: ✅
🌳 Tree 5
Vote: ✅
⬇️
Final Prediction: ✅ (4 votes vs 1 vote)

Key Concepts:

  • Bagging: Each tree trained on random sample of data (with replacement)
  • Feature Randomness: Each split considers only random subset of features
  • Voting: Classification = majority vote; Regression = average

Python Implementation

Real Example: Medical Diagnosis
  • Input: Patient symptoms, age, medical history, test results
  • 100 trees each analyze the data differently
  • Each tree votes: Disease A, Disease B, or Healthy
  • Final diagnosis based on majority vote
  • More robust than single decision tree
💡 Why Random Forest > Single Decision Tree:
  • ✅ More accurate (wisdom of the crowd)
  • ✅ Less likely to overfit
  • ✅ Handles missing data better
  • ✅ Provides feature importance ranking
  • ❌ Slower to train and predict
  • ❌ Harder to interpret than single tree
